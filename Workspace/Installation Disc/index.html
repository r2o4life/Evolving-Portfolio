How to Use This Prompt

Think of this prompt as an operating system installation disk for your AI code editor. It overrides the LLM's default instinct to start mindlessly writing code and forces it to become a structured software architect.

01

The Initial Injected Boot

Start a completely fresh session or project workspace. Paste the entire system prompt above into the chat box (or assign it to your agent system rules).

What the AI will do: It will automatically create the .blink/ folder and the markdown/JSON files in your workspace, print a clean confirmation receipt, and stop. It will ask you for your business objectives or high-level PRD.
02

Feeding the Pipeline

Instead of pasting your entire PRD into the chat window, open the newly created .blink/prd_directive.md file in your editor and paste your product requirements directly into it. Tell the AI chat: "I have populated the semantic layer files. Proceed."

Why this matters: The chat window stays completely clean. The LLM reads the file as an external configuration asset, keeping its direct chat memory free of bloat. Next, repeat this process for your technical stack inside stack_spec.json.
03

Normal Engineering Mode

Once the AI tells you the pipeline is locked, you can start coding like normal (e.g., "Build the authentication layout component"). The AI will pull instructions from the .blink/ files behind the scenes as its guardrails.

04

Handling Runtime Drifts

If you ask for a feature that contradicts your tech stack (or if the AI realizes a library you chose can't actually support a feature in your PRD), it will halt mid-track. You will see the visual # SIDE DIRECTIVE block isolate itself from the code.

Simply type your decision in plain text (e.g., "Change the spec to use 2D canvas instead of 3D"). The AI will automatically update the .blink/ files to match your new decision and cleanly resume writing your code without corrupting your repository history.
